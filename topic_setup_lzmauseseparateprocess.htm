[Setup]: LZMAUseSeparateProcess

Возможные значения:
yes, no, или x86

Значение по умолчанию:
no
Описание:

Управляет выполнением сжатия LZMA в основном процессе компилятора или в отдельном процессе.

Использование отдельного процесса для сжатия LZMA позволяет компрессору выделять большие объемы памяти, которые позволяют задать значение повыше для LZMADictionarySize и LZMANumBlockThreads, которые будут использоваться. Дополнительно, на 64-разрядном Windows (x64), может наблюдаться маленькое увеличение скорости сжатия.

На 64-разрядном Windows (x64) нет никаких ограничений на объем памяти используемый компрессором, поскольку компрессор работает в собственном 64-разрядном процессе. На 32-разрядном Windows, однако, из-за ограничений адресного пространства, обычно только приблизительно 1.5 Гбайт доступны для использования компрессором.

Значение yes включает использование 64-разрядного процесса на 64-разрядном Windows (x64) и 32-разрядного процесса на 32-разрядном Windows.

Значение x86 включает использование только 32-разрядного процесса (обычно полезно для отладки).

Значение no отключает использование отдельного процесса для сжатия LZMA.

Отметьте, что эта директива только влияет на сжатие файлов, определенных в секции [Files]; сжатие внутренних структур инсталлятора всегда выполняется в основном процессе компилятора.
