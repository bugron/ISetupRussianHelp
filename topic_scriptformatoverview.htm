Структура скрипта

Скрипт Inno Setup разделен на секции. Каждая секция отвечает за свой участок инсталляции. Секции начинаются с определением их имени, включенного в квадратные скобки ([]). В каждом разделе может быть любое число строк.

Существует два главных типа секций: секции типа [Setup] включают в себя список директив и их значений (в форме Директива=Значение) и секции типа [Files], которые включают в себя список параметров.

Примеры двух видов секций:

[Setup]
AppName=My Program

[Files]
Source: "MYPROG.EXE"; DestDir: "{app}"

Обратите внимание, что можно указать несколько секций с одним и тем же названием.

В скрипт можно добавить комментарий, который будет игнорирован при компиляции. Для этого в начале строки ставится точка с запятой (";"). Например:

; Это комментарий

Директива #include позволяет взять значение из отдельного файла и вставить его в скрипт в том месте, где она вызывается. Синтаксис директивы следующий:

#include "filename.txt"

Если имя файла указано не полностью, компилятор будет искать его в каталоге, указанном директивой #include. Если имя файла имеет префикс "compiler:", компилятор будет искать файл в своем каталоге (каталоге компилятора).

Поддерживается директива #preproc, которая указывает, использовать встроенный препроцессор, который поддерживает только директиву #include, или использовать препроцкессор Inno Setup (ISPP), который поддерживает множество других директив. Синтаксис директивы следующий:

#preproc builtin
#preproc ispp

По умолчанию, при возможности скрипты используют ISPP, а .isl файлы встроенный препроцессор.
