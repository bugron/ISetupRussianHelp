Параметры в секциях

Все секции скрипта, кроме [Setup], [Messages], [CustomMessages], [LangOptions] и [Code] состоят из строк, каждая из которых представляет собой список параметров. Рассмотрим пример секции [Files]:

[Files]
Source: "MYPROG.EXE"; DestDir: "{app}"
Source: "MYPROG.CHM"; DestDir: "{app}"
Source: "README.TXT"; DestDir: "{app}"; Flags: isreadme

Каждый параметр состоит из имени, после которого ставится двоеточие и значения. Как уже говорилось, если значение параметра не указано, он автоматически принимает значение по умолчанию. Параметры разделяются точкой с запятой и могут быть указаны в произвольном порядке.

Имя параметра заключается в двойные кавычки ("), если его значение определено пользователем (это может быть имя файла). Вообще кавычки не обязательны, но отказавшись от них, появляется риск ввода лишних символов, таких как двойные кавычки и двоеточия.

Для ввода двойных кавычек, их следует прописать дважды. Например, строку

"This "" contains "" embedded "" quotes"

компилятор видит как

This " contains " embedded " quotes

Если требуется вывести просто двойные кавычки, в скрипте их надо прописать четыре штуки: """". Здесь крайние кавычки служат для того, чтобы показать границы строки, а внутренние собственно выводятся на экран.
