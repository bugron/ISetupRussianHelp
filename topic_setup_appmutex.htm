[Setup]: AppMutex

Описание:

Эта директива используется для предотвращения установки новой версии программы, если программа выполняется в данный момент, и удаления запущенного в данный момент приложения. При указании имен одного и более ограничителей (множественные ограничители разделяются запятыми), они будут проверяться инсталлятором и деинсталлятором. Если ограничители присутствуют, инсталлятор/деинсталлятор выдаст сообщение: "[Setup или Uninstall] обнаружил, что [AppName] выполняется. Пожалуйста, завершите работу приложения, а потом нажмите "ОК" для продолжения или Cancel для отмены".

Использование этой директивы требует добавление в вашу программу кода, который создает при запуске такой ограничитель с именем, которое вы указываете в директиве. Примеры создания на Delphi, C и VB указаны ниже. Код должен выполняться в течении установки приложения.

Delphi:

CreateMutex(nil, False, 'MyProgramsMutexName');

C:

CreateMutex(NULL, FALSE, "MyProgramsMutexName");

Visual Basic (написал Peter Young):

'Place in Declarations section:
Private Declare Function CreateMutex Lib "kernel32" _
        Alias "CreateMutexA" _
       (ByVal lpMutexAttributes As Long, _
        ByVal bInitialOwner As Long, _
        ByVal lpName As String) As Long

'Place in startup code (Form_Load или Sub Main):
CreateMutex 0&, 0&, "MyProgramsMutexName"

Нет необходимости удалить ограничитель при окончании работы приложения, система сделает это автоматически. Рекомендуется делать именно так, потому что в идеале ограничитель должен существовать до тех пор, пока процесс установки/удаления не завершится полностью.

Обратите внимание, что Windows чувствителен к совпадению имен ограничителей.

Чтобы определить взаимоисключающее имя, содержащее запятую, закончите его обратный слешом.

Для справки обратитесь к разделу CreateMutex файла MS SDK help.

Например:
AppMutex=MyProgramsMutexName
