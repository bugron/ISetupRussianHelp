Inno Setup Preprocessor: Введение

Inno Setup Preprocessor (ISPP) - дополнение препроцессора для Inno Setup.

Основная цель ISPP состоит в том, чтобы автоматизировать задачи во время компиляции и уменьшить вероятность опечаток в Ваших скриптах. Например, можно объявить переменную ISPP (переменная времени компиляции) - например имя приложения и затем использовать его значение в нескольких местах Вашего скрипта. Если по некоторым причинам Вы должны поменять имя своего приложения, необходимо изменить значение только однажды в Вашем скрипте. Без ISPP Вы должны были бы, вероятно, изменить эти значения везде, где оно должно использоваться (директивы AppName, AppVerName, DefaultGroupName и т.д. секции [Setup]).

Другой пример использования ISPP - собрание информации о версии из Вашего приложения, путем прочтения информации о версии EXE файла для использования его в директивах секции [Setup] или где-либо еще. Без ISPP необходимо бы изменить свой скрипт каждый раз при изменении версии Ваших приложений.

Кроме того, с помощью условного выражение исключения возможно исключить или наоборот выполнить части скрипта: можно создать один единственный скрипт для различных версий/уровней Ваших приложений (например, испытание триального против полностью функционального).

Наконец, ISPP позволяет разделить длинные линии, используя символ разделения строки.

Обратите внимание: ISPP работает исключительно во время компиляции, и не имеет никакой функциональности во время выполнения.
