[Setup]: AppId

Значение по умолчанию:
AppName
Описание:

Значение AppId сохраняется в лог файле деинсталлятора (unins???.dat) и проверяется при последующих установках, чтобы определить, может ли это append to a particular existing uninstall log. Инсталлятор добавит новые данные в журнал удаления, если AppId существующего журнала будет тем же самым AppId текущей установки. Для практического примера, скажем, у Вас есть два инсталлятора - один My Program и другой My Program 1.1 Update. Чтобы My Program 1.1 Update смог добавлять данные в журнал My Program, необходимо установить AppId в одно и то же значение в обоих инсталляторах.

AppId также определяет текущее имя ключа реестра деинсталлятора, к которому в конце Inno Setup добавляет "_is1" . (Поэтому, если AppId будет "MyProgram", то ключ будет "MyProgram_is1".)

AppId нигде не отображается. Значение может включать константы.

Если Вы используете константу {code:..}, чтобы позволить Вашему пользователю настраивать AppId, Вы не должны возвращать реальное значение непосредственно перед тем, как установка не начнется: при необходимости можно возвращать пустое или универсальное значение. Если данная директива установлена, то инсталлятор попытается восстановить предыдущие настройки установки, а если пуста, то ничего не происходит.

Длина AppId со всеми значениями констант не должна превышать 127 символов.

Например:
AppId=MyProgram
