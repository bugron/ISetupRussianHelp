Параметры командной строки инсталлятора

Программа-инсталлятор поддерживает приведенные ниже параметры командной строки. Они пригодятся системным администраторам и другим программам, вызывающим инсталлятор.

/HELP, /?

Показывает информацию данной страницы. Игнорируется, если директива UseSetupLdr секции [Setup]равна no.

/SP-

Отключает уведомление "Эта программа установит... Вы хотите продолжить?" инсталлятора. Выполнится только если директива DisableStartupPrompt секции [Setup] равна no.

/SILENT, /VERYSILENT

Установка в тихом или очень тихом режиме. При установке в тихом режиме окно мастера и фоновое окно мастера не отображаются, отображается только окно прогресса. При очень тихом установке окно прогресса тоже не отображается. Остальная установка проходит как обычно - отображаются сообщения об ошибках в ходе установки, страница приветствия (если ее отображение не отменено директивой DisableStartupPrompt или параметром командной строки '/SP-').

Если после установки необходима перезагрузка, команда '/NORESTART' не использована и установка проходит в тихом режиме, то появится сообщение "Перезагрузить компьютер?". Если режим установки очень тихий, система перезагрузится автоматически.

/SUPPRESSMSGBOXES

Дает инсталлятору команду подавлять окна сообщения. Можно использовать только в комбинации с '/SILENT' и '/VERYSILENT'.

Ответ по умолчанию в ситуациях, когда есть выбор:

  • Да в ситуации 'Оставить новый файл?'.
  • Нет в ситуации 'Файл существует, подтверждение перезаписи.'.
  • Прервать в ситуации Прервать/Повтор.
  • Отмена в ситуации Повтор/Отмена.
  • Да (=продолжить) в ситуациях DiskSpaceWarning/DirExists/DirDoesntExist/NoUninstallWarning/ExitSetupMessage/ConfirmUninstall.
  • Да (=перезапустить) в ситуациях FinishedRestartMessage/UninstalledAndNeedsRestart.
  • Рекомендуемый выбор в ситуации PrivilegesRequiredOverridesAllowed=dialog.

Нельзя подавить следующие 5 окна сообщений:

  • Окно сообщения 'О программе'.
  • Окно сообщения 'Выход из программы установки'.
  • Окно сообщения FileNotInDir2, показываемое при необходимости смены диска и если этот диск не найден.
  • Любое окно сообщения (об ошибке), показываемое перед тем как инсталлятор (или деинсталлятор) сможет прочесть параметры командной строки.
  • , .
  • Любое диалоговое окно задач или окно сообщения показываемое встроенными функциями TaskDialogMsgBox и MsgBox.
/ALLUSERS

Указывает инсталлятору запустить инсталляцию в режиме административной установки. Действителен только когда директива PrivilegesRequiredOverridesAllowed секции [Setup] позволяет перезаписать commandline.

/CURRENTUSER

Указывает инсталлятору запустить инсталляцию в режиме неадминистративной установки. Действителен только когда директива PrivilegesRequiredOverridesAllowed секции [Setup] позволяет перезаписать commandline.

/LOG

Указывает инсталлятору создать log-файл в папке TEMP для подробного отчета о действиях и событиях при установке, обработке записей секции [Run], предпринятых в ходе установки. Применяется при отслеживании ошибок. Например, если вы подозреваете, что файл не был заменен, хотя должен был, log-файл сообщит, что файл был действительно пропущен и почему это произошло.

Log-файлу присваивается уникальное имя на базе текущей даты. (Существующие файлы не перезаписываются и не дополняются.)

Информация, содержащаяся в log-файле, техническая и поэтому не может быть понята пользователем. Формат файла может измениться без уведомления.

/LOG="filename"

Параметр похож на /LOG, но в отличие от него позволяет указать фиксированный путь/имя файла для log-файла. Если файл с указанным именем уже существует, он будет заменен. Если файл не может быть создан, установка прервется и выдаст сообщение об ошибке.

/NOCANCEL

Не дает пользователю прервать установку, отключая кнопку Отменить и игнорируя клики на кнопке Закрыть. Полезно использовать в комбинации с '/SILENT' или '/VERYSILENT'.

/NORESTART

Указывает инсталлятору не перезагружать компьютер после удачной установки. Обычно используется вместе с '/SILENT' или '/VERYSILENT'.

/RESTARTEXITCODE=код выхода

Задает инсталлятору код, который он должен возвратить, если необходима перезагрузка.  (По умолчанию, в этом случае возвращается 0.) Обычно используется с '/NORESTART'. См. также Коды выхода инсталлятора.

/CLOSEAPPLICATIONS

Указывает инсталлятору при возможности закрыть приложения, которые используют файлы, нуждающихся в замене при установке.

/NOCLOSEAPPLICATIONS

Препятствует тому, чтобы инсталлятор закрыл приложения, использующие файлы, которые должны быть обновлены при установке. Если также используется /CLOSEAPPLICATIONS, данный параметр игнорируется.

/FORCECLOSEAPPLICATIONS

Указывает инсталлятору закрывать приложения принудительно.

/NOFORCECLOSEAPPLICATIONS

Запрещает инсталлятору закрывать приложения принудительно. Если еще и использовать параметр командной строки /FORCECLOSEAPPLICATIONS, то данный параметр игнорируется.

/LOGCLOSEAPPLICATIONS

Указывает инсталлятору вести дополнительный журнал событий при закрытии приложений для отладочных целей.

/RESTARTAPPLICATIONS

Указывает инсталлятору при возможности перезапустить приложения.

/NORESTARTAPPLICATIONS

Препятствует тому, чтобы инсталлятор перезапускал приложения. Если также используется /RESTARTAPPLICATIONS, данный параметр игнорируется.

/LOADINF="filename"

Указывает инсталлятору загрузить параметры установки из указанного файла после проверки командной строки. Этот файл может быть создан с помощью команды '/SAVEINF='.

Не забудьте ставить кавычки при использовании имен, содержащих пробелы.

/SAVEINF="filename"

Указывает инсталлятору сохранить параметры установки в указанный файл.

Не забудьте ставить кавычки при использовании имен, содержащих пробелы.

/LANG=language

Задает используемый язык. language указывает имя языка, заданного в параметре секции [Languages]. При использовании параметра /LANG диалог Выбор языка будет подавлен.

/DIR="x:\dirname"

Меняет имя папки, заданное по умолчанию, на странице Выбор папки назначения. Должен быть задан полный путь. Может содержать префикс "expand:", который указывает инсталлятору подставить значения вместо констант в имени. Например: '/DIR=expand:{autopf}\My Program'.

/GROUP="folder name"

Меняет имя папки на странице Выбор папки меню Пуск. Может содержать префикс "expand:", см. '/DIR='. Если директива DisableProgramGroupPage секции [Setup] равна yes, этот параметр командной строки игнорируется.

/NOICONS

Указывает инсталлятору установить флажок Не создавать значков на странице Выбор папки меню Пуск.

/TYPE=type name

Переопределяет значение типа установки.

Если указанный тип существует и не является пользовательским типом, то параметр /COMPONENTS будет проигнорирован.

/COMPONENTS="список имен компонентов, разделенных запятыми"

Меняет выбранные по умолчанию компоненты. В этом случае параметр командной строки указывает инсталлятору менять тип установки на Выборочный. Если пользовательский тип не определен, этот параметр игнорируется.

Только указанные компоненты будут выбраны; остальные останутся невыбранными.

Если имя компонента начинается символом "*", все дочерние компоненты также будут выбраны (за исключением тех, которые имеют флаг dontinheritcheck). Если имя компонента начинается символом "!", выбор компонента будет отменен.

Этот параметр не изменяет состояние компонентов, которые имеют флаг fixed.

Например:
Отменяется выбор всех компонентов, потом выбираются компоненты "help" и "plugins":
/COMPONENTS="help,plugins"
Например:
Отменяется выбор всех компонентов, потом выбирается родительский компонент, кроме дочериного компонента child:
/COMPONENTS="*parent,!parent\child"
/TASKS="список имен задач, разделенных запятыми"

Определяет список задач, которые должны быть изначально выбраны.

Только указанные задачи будут выбраны; остальные останутся невыбранными. Если вы хотите оставить на месте задачи по умолчанию или выбрать/отменить только некоторые из них, используйте параметр /MERGETASKS.

Если имя задачи начинается символом "*", все дочерние задачи также будут выбраны (за исключением тех, которые имеют флаг dontinheritcheck). Если имя задачи начинается символом "!", ее выбор будет отменен.

Например:
Отменяется выбор всех задач, потом выбираются задачи "desktopicon" и "fileassoc":
/TASKS="desktopicon,fileassoc"
Например:
Отменяется выбор всех задач, потом выбирается родительская задача, кроме дочериного:
/TASKS="*parent,!parent\child"
/MERGETASKS="список имен задач, разделенных запятыми"

Подобен параметру /TASKS, за исключением того, что указанные задачи будут объединены с задачами, которые в обратном случае были бы выбраны по умолчанию.

Если UsePreviousTasks имеет значение yes, то указанные задачи будут выбраны или их выбор будет отменен после восстановления всех предыдущих задач.

Например:
Оставить по умолчанию выбранные задачи, но дополнительно выбрать задачи "desktopicon" и "fileassoc":
/MERGETASKS="desktopicon,fileassoc"
Например:
Оставить по умолчанию выбранные задачи, но отменить выбор задачи "desktopicon":
/MERGETASKS="!desktopicon"
/PASSWORD=password

Задает пароль. Если не указана директива Password секции [Setup], этот параметр игнорируется.

Если введен неправильный пароль, параметр также игнорируется.
