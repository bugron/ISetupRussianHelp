Inno Setup Preprocessor: #pragma

Синтатсис

pragma-directive: <pragma-option>

<pragma-itokens>

<pragma-msg>

<pragma-verblev>

<pragma-include>

<pragma-spansymb>
pragma-option: pragma (option | parseroption) - <letter> (+ | -) [- <letter> (+ | -) ]...
pragma-itokens: pragma (inlinestart | inlineend) <expr>
pragma-msg: pragma (message | warning | error) <expr>
pragma-verblev: pragma verboselevel <expr>
pragma-include: pragma include <expr>
pragma-spansymb: pragma spansymbol <expr>

Описание

pragma является специальной директивой. Пожалуйста, отметьте, что, если ISPP не в состоянии проанализировать параметры этой директивы (из-за опечатки или неправильного синтаксиса), ошибка не произойдет - будет выведено только предупреждение; это делается для совместимости с другими препроцессорами, у которых может быть их собственный синтаксис директивы pragma.

Первый синтаксис директивы pragma управляет опциями, который ISPP использует, чтобы считать источник. Существуют две группы опций. Каждая группа состоит из 26 флагов. Каждым флагом, имеет присвоенную латинскую букву. Вы определяете опции, вводя название группы (option или parseroption), тогда буква находится после тире. После буквы должны быть определены знак "плюс" или "минус". Знак "плюс" включает опцию, "минус" - выключает. Неограниченное число вариантов может быть определено сразу (см. синтаксис). Список опций приведен в конце этой темы.

Первая группа опций (option) управляет опциями целого механизма ISPP, в то время как вторая группа (parseroption) управляет опциями, определенными для синтаксического анализатора. Список опций приведен в конце этой темы.

Второй синтаксис используется, чтобы определить встроенные направляющие разделители: запуск и окончание, соответственно. После ключевого слова описания (inlinestart или inlineend) должно следовать выражение строкового типа. Она не должна быть пустая. Из строки берутся только первые семь символов .Позволяется определить тот же самый маркер и для запускающийся и для конечных разделителей. По умолчанию, {# (открытие фигурной скобки и знака числа) и } (закрывающая фигурная скобка) принимаются.

Третий синтаксис директивы pragma выпускает сообщение типа, определенного ключевым словом после имени директивы. Сообщения и предупреждения отправляются окну сообщений компилятора. Ошибки показываются (компилятором) в месседжбоксах. Выражение должно иметь строковый тип.

Четвертый синтаксис устанавливает уровень многословия. Когда многословный режим включен (см. ниже), данный синтаксис управляет важным уровнем сообщений. Наименее важные сообщения обнаружатся только тогда, когда самый высокий многословный уровень (9) будет установлен.

Пятый синтаксис устанавливает путь include. Выражение может определить множество путей разделенные точкой с запятой. Список каталогов, в которых ISPP пытается найти файлы, упомянуты в директиве include.

Последний синтаксис устанавливает символ, разделяющий длинные строки. Он не должен быть пустым. Берется только первый символ в строке.

Опции ISPP

c Указывает, что перевод должен быть отправлен компилятору после того, как предварительная обработка завершена. Состояние по умолчанию: вкл.
e Определяет, должны ли пустые строки из источника быть испущены в перевод, так же как строки с директивами ISPP должны быть заменены пустыми строками. Состояние по умолчанию: выкл.
v Включает или отключает многословный режим. Состояние по умолчанию: выкл.

Опции синтаксического анализатора

b Short-circuit boolean evaluation. Состояние по умолчанию: вкл.
m Short-circuit multiplication evaluation. (In "0 * A", A will not be evaluated, since the result of expression is known to be zero.) Состояние по умолчанию: выкл.
p Строковые литералы стиля Паскаля. В отключенном состоянии использует строковые литералы C-стиля (с escape-последовательностями). Состояние по умолчанию: вкл.
u Позволить необъявленные идентификаторы. Если встречается неопределенный идентификатор , то ISPP выводит ошибку, если данная опция не будет включена, иначе неопределенный идентификатор (тот, который не похож на вызов функции) будет считаться пустым значением. Состояние по умолчанию: выкл.

Примеры

#pragma parseroption -b- -u+
#pragma option -c-
#pragma warning "Variable value is: " + MyVar
#pragma option -v+
#pragma verboselevel 9
#pragma inlinestart "$("
#pragma inlineend ")"
#pragma include __INCLUDE__ + ";D:\INCLUDE"
#pragma spansymbol "_"
