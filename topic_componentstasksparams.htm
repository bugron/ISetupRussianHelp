Параметры секций [Components] и [Tasks]

Существуют два параметра, которые поддерживаются всеми секциями, содержащими параметры, кроме [Types], [Components] и [Tasks]. Это:

Components

Список имен компонентов, разделенных пробелами, которым компилятору указывается, к какому из компонентов относится запись. Запись будет обработана, если пользователь выберет компонент, в который он включен (например, установится файл).

Запись без параметра Components будет обработана всегда, если только другой параметр этого не запретит.

Например:
[Files]
Source: "MYPROG.EXE"; DestDir: "{app}"; Components: main
Source: "MYPROG.CHM"; DestDir: "{app}"; Components: help
Source: "README.TXT"; DestDir: "{app}"
Tasks

Список имен заданий, разделенных пробелами, которым компилятору указывается, к какому из заданий относится запись. Запись будет обработана, если пользователь выберет задание, в которое он включен (например, установится файл).

Запись без параметра Tasks будет обработана всегда, если только другой параметр этого не запретит.

Обратите внимание, что флажок Не создавать папку в меню «Пуск» на странице Выберите папку в меню «Пуск» не влияет на записей секции [Icons], которые имеют параметры Tasks, так как они имеют свои собственные флажки.

Например:
[Icons]
Name: "{group}\My Program"; Filename: "{app}\MyProg.exe"; Components: main; Tasks: startmenu
Name: "{group}\My Program Help"; Filename: "{app}\MyProg.chm"; Components: help; Tasks: startmenu
Name: "{commondesktop}\My Program"; Filename: "{app}\MyProg.exe"; Components: main; Tasks: desktopicon


Помимо разделенных пробелом списков, Вы можете также использовать логические выражения в качестве параметров Components и Tasks. Поддерживаются операторы not, and и or. Например:

[Components]
Name: a; Description: a
Name: b; Description: b

[Tasks]
Name: p; Description: a or b; Components: a or b
Name: q; Description: a and b; Components: a and b
Name: r; Description: not a or b; Components: not a or b
Name: s; Description: not (a or b); Components: not (a or b)
Name: t; Description: a or b - old style; Components: a b
