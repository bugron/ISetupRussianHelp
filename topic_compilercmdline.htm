Командная строка инсталлятора

  • Скрипты также могут быть скомпилированы из командной строки. Общий вид использования командной строки:

    compil32 /cc <имя скрипта>

    Например:
    compil32 /cc "c:\isetup\samples\my script.iss"

    Как показано в приведенном выше примере, имена файлов, включающих пробелы, должны заключаться в кавычки.

    При запуске компилятора из командной строки отображение обычного хода выполнения или сообщений об ошибках остается тем же. Компилятор вернет код выхода: 0 при успешной компиляции; 1, если параметры командной строки были указаны неверно; 2, если компиляция завершилась неудачно.

  • Также компилировать скрипты можно в консольном режиме, ISCC.EXE. Общий вид использования:

    iscc [параметры] <имя скрипта>

    Или прочитать из стандартного ввода:

    iscc [параметры] -

    Например:
    iscc "c:\isetup\samples\my script.iss"

    Как показано в приведенном выше примере, имена файлов, включающих пробелы, должны заключаться в кавычки.

    Допустимые параметры: "/O-" отключает вывод (перезаписывается значение директивы Output), "/O+" включает вывод (перезаписывается значение директивы Output), "/O" - определяет выходной путь (перезаписывается значение директивы OutputDir), "/F" - определяет конечное название файла (перезаписывается значение директивы OutputBaseFilename), "/S" - определяет инструмент подписи, "/Q[p]" - определяет тихую компиляцию (Вы увидите только сообщения об ошибках, параметр "p" показывет прогресс), и "/?" - показать справку.

    Например:
    iscc /Qp /O"My Output" /F"MyProgram-1.0" /Sbyparam=$p "c:\isetup\samples\my script.iss"

    ISCC вернет код выхода: 0 при успешной компиляции; 1, если параметры командной строки были указаны неверно или произошла ошибка; 2, если компиляция завершилась неудачно.

  • Мастер скрипта инсталлятора можно также запустить из командной строки. Использование командной строки имеет следующий вид:

    compil32 /wizard <имя мастера> <имя скрипта>

    Например:
    compil32 /wizard "MyProg Script Wizard" "c:\temp.iss"

    Как показано в приведенном выше примере, имена файлов и мастеров, включающих пробелы, должны заключаться в кавычки.

    При запуске мастера из командной строки отображение сообщений об ошибках остается тем же. Мастер скрипта инсталлятора вернет следующий код выхода: 0, если нет никаких ошибок, в этом случае он сохранит скрипт под заданным именем; 1, если параметры командной строки были указаны неверно; 2, если полученный скрипт сохранить не удался. Если пользователь прервет работу мастера, будет возвращен код выхода 0 и скрипт не сохранится.
