[Setup]: LZMANumBlockThreads

Возможные значения:
1 до 32

Значение по умолчанию:
1
Описание:

Сжимая большой объем данных, у компрессора LZMA2 есть возможность разделить данные на "блоки" и сжать два или больше из этих блоков параллельно с помощью дополнительных потоков (если достаточное питание процессора доступно). Эта директива определяет число потоков, которые компрессор LZMA2 может сжать параллельно.

Память, требуемая для сжатия, когда используются многократные блочные потоки, примерно равна:

LZMANumBlockThreads * (Нормальное использование памяти + (LZMABlockSize * 2))

Так как LZMA2 (и LZMA) по умолчанию использует два потока для обнаружения соответствия (см. CompressionThreads), по идеалу должно быть два ядра процессора, доступные для каждого потока блоков. Таким образом, чтобы увидеть максимальное преимущество от значении 2, необходимы четыре ядра.

Деление данных в многократные блоки может уменьшить степень сжатия, поскольку компрессор не может найти соответствия через блоки. Использование большого размер блока может помочь смягчить это.

Если выдается ошибка "Out of memory", тогда многократные блочные потоки включены в комбинации с уровнем сжатия, который использует большой размер словаря (такой как ultra64), возможно, должен быть установлен LZMAUseSeparateProcess.
