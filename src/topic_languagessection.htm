Секция [Languages]

Inno Setup поддерживает многоязыковые установки. Секция [Languages] определяет языки, доступные в программе-инсталляторе.

Инсталлятор определяет язык по умолчанию, который будет использоваться в сообщениях, в следующем порядке:

Не Unicode Inno Setup:

  1. Ищет язык, параметр LanguageID которого (обычно указанный в секции [LangOptions] в .isl файле) совпадает с идентификаторами основного и дополнительного языка пользовательского интерфейса или локализации (зависит от значения директивы LanguageDetectionMethod) и чья LanguageCodePage установлена в соответствии с системной кодовой страницей.
  2. Если совпадений не найдено, ищет совпаденя с идентификатором и кодовой страницей только основного языка. Если два и более доступных языка имеют одинаковые идентификаторы и кодовые страницы, применяется первый в списке секции [Languages].
  3. Если совпадений не найдено, первый указанный язык в секции [Languages] принимается языком по умолчанию.

Unicode Inno Setup:

  1. Ищет язык, параметр LanguageID которого (обычно указанный в секции [LangOptions] в .isl файле) совпадает с идентификаторами основного и дополнительного языка пользовательского интерфейса или локализации (зависит от значения директивы LanguageDetectionMethod).
  2. Если совпадений не найдено, ищет совпаденя с идентификатором только основного языка. Если два и более доступных языка имеют одинаковые идентификаторы, применяется первый в списке секции [Languages].
    Исключение: Упрощенный китайский исключается из рассмотрения в этом шаге, если язык UI пользователя или локальный язык (в зависимости от установки LanguageDetectionMethod) является Традиционным китайским и наоборот.
  3. Если ни один не найден, первый указанный язык в секции [Languages] принимается языком по умолчанию.

Если директива ShowLanguageDialog секции [Setup] будет установлена в yes (значение по умолчанию), то диалоговое окно Выбор языка будет выведено на экран, который дает пользователю возможность выбрать язык инсталлятора. На не Unicode версии Inno Setup будут скрыты языки, которые не могут быть выведены на экран должным образом на системе пользователя. См. секцию [LangOptions] для справки.

Пример секции [Languages] (указывает два языка: английский, расположенный в стандартном файле Default.isl и датский).

[Languages]
Name: "en"; MessagesFile: "compiler:Default.isl"
Name: "nl"; MessagesFile: "compiler:Languages\Dutch.isl"
Name  (Обязательно)

Имя языка, можно задать любое. Можно использовать как префикс в секциях [LangOptions] или [Messages], чтобы параметры применялись только к одному языку. Константа {language} содержит имя выбранного языка.

Например:
Name: "en"
MessagesFile  (Обязательно)

Указывает имя (имена) файла (файлов), из которых следует считывать стандартные сообщения. Этот файл должен быть расположен в исходной папке скрипта, если только для него не указан полный путь или нет префикса "compiler:" - в этом случае компилятор будет искать указанный файл в своем каталоге.

Когда указаны несколько файлов, они считываются в том порядке, в котором указаны, таким образом последний файл сообщений перекрывает сообщения из предыдущих файлов.

Чтобы узнать больше о файлах .isl, обратитесь к разделу справки секция [Messages].

Примеры:
MessagesFile: "compiler:Dutch.isl"
MessagesFile: "compiler:Default.isl,compiler:MyMessages.isl"
LicenseFile

Определяет имя файла лицензионного соглашения в формате .txt или .rtf, который отобразится перед тем, как пользователю будет предложено выбрать папку назначения для устанавливаемой программы. Этот файл должен быть расположен в исходной папке скрипта, если только для него не указан полный путь или нет префикса "compiler:" - в этом случае компилятор будет искать указанный файл в своем каталоге.

Например:
LicenseFile: "license-Dutch.txt"
InfoBeforeFile

Определяет имя файла ReadMe в формате .txt или .rtf, который отобразится перед тем, как пользователю будет предложено выбрать папку назначения для устанавливаемой программы. Этот файл должен быть расположен в исходной папке скрипта, если только для него не указан полный путь или нет префикса "compiler:" - в этом случае компилятор будет искать указанный файл в своем каталоге.

Например:
InfoBeforeFile: "infobefore-Dutch.txt"
InfoAfterFile

Определяет имя файла ReadMe в формате .txt или .rtf, который отобразится после успешной установки программы. Этот файл должен быть расположен в исходной папке скрипта, если только для него не указан полный путь или нет префикса "compiler:" - в этом случае компилятор будет искать указанный файл в своем каталоге.

Отличие от файлов с флагом isreadme заключается в том, что эти файлы отобразятся на страницах мастера установки, а не в окне Блокнота.

Например:
InfoAfterFile: "infoafter-Dutch.txt"
