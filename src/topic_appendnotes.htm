Обновление журнала деинсталлятора

Когда устанавливается новая версия приложения, вместо того, чтобы создать новый файл журнала удаления (unins???.dat), инсталлятор по умолчанию ищет и добавляет записи к существующему файлу, который принадлежит старому приложению в той же самой папке. Таким образом, когда приложение будет удалено, изменения, произведенные всеми инсталляторами, будут отменены.

Деинсталлятор будет использовать сообщения от самого последнего инсталлятора. Однако есть исключение: если инсталлятор был создан более старой версией Inno Setup с более старой версией деинсталлятора, чем в системе пользователя, ни существующий деинсталлятор, ни его сообщения не будут заменены. В этом случае файл журнала удаления будет все же добавлен, так как формат файла обратно совместим.

Имя приложения, выведенное на экран в деинсталляторе, будет тем же самым значением, которая была определена в директиве AppName секции [Setup], если UpdateUninstallLogAppName равен no.

Возможность добавления записей к существующему журналу деинсталлятора нововведение с Inno Setup 1.3. Если Вы хотите отключить его, установите директиву UninstallLogMode в секции [Setup].

Примечание: Инсталлятор может добавить файл журнала удаления, которые создавались только Inno Setup 1.3.1 (или позднее).
