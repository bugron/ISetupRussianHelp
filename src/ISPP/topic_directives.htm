Inno Setup Preprocessor: Директивы

В ISPP директивы могут быть использованы двумя методами: простым и встроенным.

Простой метод заключается в том, что строка с директивой начинается с символа "решётки" (#). Например, установим переменную с именем MyAppName и присвоим ей значение "My Program":

#define MyAppName "My Program"

Встроенный метод позволяет подключить директиву, которая была установлена ранее. В этом случае директива записывается между начальными символами {# (открывающая фигурная скобка плюс решётка) и завершающим символом } (закрывающая фигурная скобка). Например, присвоим директиве Inno Setup с именем AppName значение ранее установленной переменной MyAppName (см. пример выше):

[Setup]
  AppName={#MyAppName}

При выполнении сценарии вместо выражения {#MyAppName} будет подставлено значение "My Program" из переменной MyAppName. Таким образом, директиве AppName будет присвоено значение "My Program".

Как видно из приведенного выше примера, не обязательно указывать имя директивы emit, если она используется встроенным методом. Поэтому выражение {#MyAppName} является сокращенной записью {#emit MyAppName}.

Доступные директивы:

  • #append
  • #define
  • #dim, #redim
  • #emit
  • #error
  • #expr
  • #file
  • #for
  • #if, #elif, #else, #endif
  • #ifdef, #ifndef, #ifexist, #ifnexist
  • #include
  • #insert
  • #pragma
  • #sub, #endsub
  • #undef
