Секция [Components]

Это не обязательная секция. Она определяет компоненты, которые инсталлятор отобразит на странице Выбор компонентов для выборочной установки.

Сама по себе эта секция ничего не делает: для того, чтобы компоненты были действительно установлены на компьютер пользователя, секция [Components] должна быть связана с другими секциями. См. Параметры секций [Components] и [Tasks].

Пример секции [Components]:

[Components]
Name: "main"; Description: "Main Files"; Types: full compact custom; Flags: fixed
Name: "help"; Description: "Help Files"; Types: full
Name: "help\english"; Description: "English"; Types: full
Name: "help\dutch"; Description: "Dutch"; Types: full

Пример создает четыре компонента: компонент "main" будет установлен, если пользователь выберет тип "full", "compact" или "custom". Компонент "help" с двумя дочерними компонентами будет установлен, если пользователь выберет тип "full". При выбранном типе "custom" можно будет выбирать компонент "help" по желанию.

Список поддерживаемых параметров:

Name (Обязательно)

Имя компонента.

Общее число \ или / в имени компонента называется уровнем компонента. Все компоненты с уровнем 1 и выше являются дочерними. Компонент, состоящий из дочернего компонента и имеющий уровень на единицу меньше, чем дочерний, называется родительским. Дочерние компоненты, имеющие одинаковый родительский компонент, называются родственными.

Дочерний компонент не может быть выбран отдельно от родительского. Родительский компонент не может быть выбран, если ни один из его дочерних компонентов не выбран, пока параметр секции Components непосредственно не ссылается на родительский компонент, или родительский компонент включает флаг checkablealone и сам по себе он ничего не устанавливает.

Если родственные компоненты имеют флаг exclusive , можно выбрать только один из них.

Например:
Name: "help"
Description (Обязательно)

Описание компонента (может содержать константы). Описание отображается в процессе установки.

Например:
Description: "Help Files"
Types

Список типов, поддерживающих данный компонент. Если пользователь при установке выберет один из поддерживаемых типов, компонент будет установлен. Типы разделяются пробелами.

Если флаг fixed (см. ниже) не установлен, любой выборочный тип установки (тип с флагом iscustom) в этом списке игнорируется инсталлятором.

Например:
Types: full compact
ExtraDiskSpaceRequired

Определяет дополнительное место на диске аналогично директиве ExtraDiskSpaceRequired секции [Setup].

Например:
ExtraDiskSpaceRequired: 0
Flags

Параметры для установки дополнительных опций. Разделяются пробелами. Поддерживаемые значения:

checkablealone

Указывает, что компонент может быть выбран, когда ни один из его дочерних компонентов не выбран. По умолчанию, если ни один параметр секции Components непосредственно не ссылается на компонент, тогда отмена выбора всех дочерних элементов компонента приведет к отмене выбора родительского компонента.

dontinheritcheck

Указывает, что компонент не выбирается автоматически при выборе родительского компонента. Не действует на компонентов, не имеющих дочерних компонентов, не комбинируется с флагом exclusive.

exclusive

Указывает инсталлятору, что из двух и более дочерних компонентов, также имеющих флаг exclusive, может быть выбран только один.

fixed

Указывает инсталлятору, что компонент с этим флагом не может быть выбран или не выбран пользователем при установке.

restart

Указывает инсталлятору, что при выборе пользователем компонента после установки следует перезагрузить компьютер, даже если в этом нет необходимости (например в секции [Files] могут находиться записи с флагом restartreplace). Действует так же, как директива AlwaysRestart, только для компонентов.

disablenouninstallwarning

Указывает инсталлятору не предупредить пользователя, что этот компонент не удалится после того, как он снимет с него флажок, если этот компонент уже установлен на данном компьютере.

В зависимости от сложности компонентов можно попробовать использовать секцию [InstallDelete] и этот флаг, для автоматического "удаления" компонентов с не выбранным флажком.

Например:
Flags: fixed
