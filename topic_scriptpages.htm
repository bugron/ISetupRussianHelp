Pascal Scripting: Использование дополнительных страниц мастера

Pascal-скрипт позволяет добавить дополнительные страницы в инсталлятор. Сюда входят "полу готовые" страницы, которыми можно дополнить стандартную последовательность страниц, и полностью произвольные страницы, в которые можно добавить любые элементы на выбор.

Для того, чтобы воспользоваться дополнительными страницами мастера, сначала их надо создать во внешней функции InitializeWizard. При этом можно воспользоваться страницами, созданными с помощью функций CreateInput...Page и CreateOutput...Page или "пустыми" страницами, созданными функцией CreateCustomPage. Эти функции описаны в разделе Встроенные функции.

После создания страниц на них добавляются элементы, для добавления каждого из которых либо вызывается специальный метод (для "полу готовых" элементов), либо вручную пишется программный код.

Большинство функций Create...Page в качестве первого параметра имеют идентификатор страницы (page ID), которая является идентификатором страницы, после которого будет помещена создаваемая страница. Существует несколько способов определить идентификатор существующей страницы. Созданные вами страницы имеют параметры идентификаторов, которые хранятся в их ID. Существующие же страницы имеют уже заданные идентификаторы. Например, страница Приветствие имеет идентификатор wpWelcome. Все существующие идентификаторы приведены в разделе Встроенные функции.

После того, как дополнительная страница создана, инсталлятор определит и отобразит ее так же, как стандартные страницы. Это относится и к вызову всех встроенных функций для страниц, таких как NextButtonClick и ShouldSkipPage.

В любой момент в ходе инсталляции вы можете извлечь значения, введенные пользователем либо с помощью специальных свойств страниц, либо с помощью свойств, созданных вами.

Для того, чтобы посмотреть пример использования страниц и встроенных функций для создания дополнительных страниц мастера, откройте файл CodeDlg.iss, который находится в папке Examples директории Inno Setup. Для того, чтобы посмотреть пример создания полностью произвольной дополнительной страницы мастера, откройте файл CodeClasses.iss.
