[Setup]: UsePreviousPrivileges

Возможные значения:

yes или no

Значение по умолчанию:

yes
Описание:

При значении yes инсталлятор при запуске посмотрит в реестре, не устанавливалась ли та же программа ранее в одном из двух режимов установки и если так, будет использован предыдущый режим установки вместо того, чтобы спросить пользователя.

Если PrivilegesRequiredOverridesAllowed не позволяет значение dialog, данная директива игнорируется.

Значение UsePreviousPrivileges должно быть no когда AppId содержит константы и PrivilegesRequiredOverridesAllowed позволяет значение dialog.
